# 개인 개발 블로그 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 프로젝트명
**개인 개발 블로그** (Personal Development Blog)

### 목적
기술 글쓰기의 효율성을 높이기 위해 Notion을 CMS로 활용하는 개인 기술 블로그 플랫폼 구축

### 배경
- Notion에서 글을 작성하면 자동으로 블로그에 반영되는 워크플로우 확보
- 별도의 관리자 페이지 없이 Notion 인터페이스로 모든 콘텐츠 관리
- 개발자 친화적인 기술 블로그 포트폴리오 구축

## 3. Notion 데이터베이스 구조

### 데이터베이스 필드 정의

| 필드명 | 필드 타입 | 설명 | 필수 여부 |
|--------|----------|------|----------|
| **Title** | Title | 글의 제목 | 필수 |
| **Category** | Select | 글의 카테고리 (React, Node.js, TypeScript 등) | 필수 |
| **Tags** | Multi-select | 글 관련 태그 여러 개 선택 가능 | 선택 |
| **Published** | Date | 글 발행 날짜 | 필수 |
| **Status** | Select | 글 상태 (초안/발행됨) | 필수 |
| **Cover Image** | Files & media | 글 썸네일 이미지 | 선택 |
| **Description** | Text | 글 짧은 설명/요약 | 선택 |
| **Content** | Page content | 글 본문 (Notion 블록 형식) | 필수 |

### Category 예시값
- React
- Next.js
- TypeScript
- Node.js
- Database
- DevOps
- Architecture

## 4. 주요 기능 명세

### 4.1 Notion 데이터베이스에서 블로그 글 목록 가져오기
**설명**: 발행됨 상태의 글을 Notion API를 통해 주기적으로 또는 온디맨드로 조회

**세부사항**:
- Published 날짜 기준 최신순 정렬
- Status = "발행됨"인 글만 표시
- 페이지네이션 지원 (10개 글 단위)
- 제목, 카테고리, 발행일, 썸네일 이미지 표시

### 4.2 개별 글 상세 페이지 표시
**설명**: Notion 글의 전체 콘텐츠를 렌더링하여 상세 페이지 제공

**세부사항**:
- Notion 블록(텍스트, 이미지, 코드블록 등) 파싱 및 렌더링
- 글 제목, 발행일, 카테고리, 태그 표시
- 목차(Table of Contents) 자동 생성 (헤딩 기반)
- 이전/다음 글 네비게이션
- 공유 버튼 (Twitter, LinkedIn, 복사)

### 4.3 카테고리별 필터링
**설명**: 특정 카테고리의 글만 조회 및 표시

**세부사항**:
- 좌측 또는 상단 네비게이션에서 카테고리 선택
- 선택한 카테고리의 글 목록만 필터링
- 현재 선택 카테고리 하이라이트 표시
- "모든 글" 옵션으로 필터 초기화

### 4.4 검색 기능
**설명**: 글 제목 또는 태그 기반 검색

**세부사항**:
- 검색 입력창에서 실시간 검색 지원
- 제목 또는 설명(Description) 텍스트 포함 여부 검색
- 검색 결과 개수 표시
- 검색 결과 없음 시 안내 메시지

### 4.5 반응형 디자인
**설명**: 모바일, 태블릿, 데스크톱 전체 기기에서 최적화된 UI

**세부사항**:
- 모바일: 스택 레이아웃, 햄버거 메뉴 네비게이션
- 태블릿: 유연한 2단 레이아웃
- 데스크톱: 3단 레이아웃 (사이드바/메인/사이드바)
- 다크모드/라이트모드 지원

## 5. 화면 구성

### 5.1 홈 페이지 (`/`)
**구성 요소**:
- 사이트 헤더 (로고, 네비게이션)
- 영웅 섹션 (블로그 소개, 프로필)
- 최신 글 목록 (카드 형식)
- 카테고리 필터 사이드바 (모바일에서는 상단 탭)
- 검색 바
- 페이지네이션

**데이터 흐름**: Notion API → 글 목록 조회 → 최대 10개 글 표시

### 5.2 글 상세 페이지 (`/blog/[id]`)
**구성 요소**:
- 글 제목, 발행일, 카테고리, 태그
- 썸네일 이미지 (선택)
- 목차 (우측 사이드바)
- 글 본문 (Notion 블록 렌더링)
- 공유 버튼
- 이전/다음 글 네비게이션
- 댓글 섹션 (선택사항)

**데이터 흐름**: URL 파라미터 id → Notion 페이지 조회 → 블록 파싱 및 렌더링

### 5.3 카테고리 페이지 (`/blog/category/[category]`)
**구성 요소**:
- 카테고리 제목
- 카테고리 설명 (선택)
- 해당 카테고리의 글 목록
- 검색 및 정렬 옵션

**데이터 흐름**: URL 파라미터 category → Notion 필터 쿼리 → 카테고리별 글 조회

### 5.4 About 페이지 (`/about`)
**구성 요소**:
- 자기소개
- 기술 스택
- 소셜 링크 (GitHub, Twitter 등)

## 8. 비기능 요구사항

### 성능
- **페이지 로딩**: Lighthouse 성능 점수 90 이상
- **이미지 최적화**: next/image 사용으로 웹사이트 포맷 및 사이징 자동 최적화
- **캐싱**: Notion 데이터 주기적 캐싱 (ISR - Incremental Static Regeneration)
- **번들 크기**: JavaScript 번들 500KB 이하

### 접근성
- WCAG 2.1 AA 준수
- 스크린리더 지원
- 키보드 네비게이션 가능
- 충분한 색상 대비

### SEO
- 동적 메타데이터 (OG 태그, 디스크립션)
- 사이트맵 및 robots.txt
- 구조화된 데이터 (Schema.org)
- 친화적인 URL 구조

### 보안
- NOTION_API_KEY는 환경 변수로만 관리 (노출 금지)
- Notion 데이터베이스 공개 여부 확인
- CORS 설정 (Vercel 배포 시)

### 확장성
- 새로운 필드 추가 용이 (필드 정의 추상화)
- 블록 타입별 렌더러 모듈화
- 다중 Notion 데이터베이스 지원 가능하도록 설계
